## 💻 API Endpoints Reference

```bash
# All endpoints below are available on:
# http://localhost:5001

GET    /health              # Health check
GET    /status              # Get machine status
POST   /power_cut           # Cut machine power
POST   /power_restore       # Restore power
POST   /reset               # System reset
POST   /event               # Log incoming event
GET    /api/docs            # API documentation
```

**Example API calls:**
```bash
# Check if server is running
curl http://localhost:5001/health

# Get current status
curl http://localhost:5001/status

# Power control
curl -X POST http://localhost:5001/power_cut \
  -H "Content-Type: application/json" \
  -d '{"machine_name": "PROTEUS_SIM"}'
```

## 📞 Getting Help

### Problem: Server won't start
**Solution:**
1. Check Flask is installed: `pip install flask`
2. Check port isn't in use: `lsof -i :5001` (Linux/Mac)
3. View error log: `tail -f proteus_simulation.log`

### Problem: API not responding
**Solution:**
1. Make sure main_proteus.py is running
2. Check port 5001 is correct
3. Try from same machine first: `localhost:5001`

### Problem: Tests failing
**Solution:**
1. Make sure simulation is running first
2. Wait 2-3 seconds for Flask to start
3. Run with verbose output: `python test_proteus_simulation.py -v`

### Problem: Can't connect from another computer
**Solution:**
1. Server binds to `0.0.0.0` - that's correct
2. Use your computer's IP: `http://192.168.x.x:5001`
3. Check firewall allows port 5001
